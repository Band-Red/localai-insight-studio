# 5. وثائق الجودة والاختبار | Quality & Testing

## 5.1 إستراتيجية الاختبار (Test Strategy)

يعتمد نظام الجودة في المشروع على ثلاثة مستويات:

1.  **الاختبار اليدوي الشامل (Manual QA):** عبر تنفيذ "جولة التجريبية" (Walkthrough) بعد كل تحديث كبير.
2.  **اختبار التكامل (Integration Testing):** التأكد من أن جسر البايثون (Python Bridge) يتواصل بشكل سليم مع واجهة Electron.
3.  **فحص التوافقية:** تجربة التطبيق على إصدارات مختلفة من نظام ويندوز (10 و 11).

---

## 5.2 دليل تشغيل الاختبارات (How to Run Tests)

حالياً، يتم فحص المشروع عبر الأوامر التالية:

1.  **فحص لب النظام:** تشغيل `npm run dev` ومراقبة الـ Console بحثاً عن أي أخطاء في الـ IPC.
2.  **فحص البناء:** تشغيل `npm run build` للتأكد من عدم وجود أخطاء في الـ Typescript أو التنسيقات.
3.  **فحص المخرجات:** التأكد من توليد ملف `.exe` سليم عبر أمر `npm run dist`.

---

## 5.3 وثائق تغطية الاختبارات (Coverage)

نظراً لطبيعة المشروع (Local-First)، نركز على تغطية الحالات التالية (Test Cases):

- [ ] نجاح اختيار الملفات عبر `dialog`.
- [ ] دقة الرسم البياني في الـ Dashboard.
- [ ] قدرة النموذج على تعديل الكود عبر الـ Visual Inspector.
- [ ] حفظ واستعادة الإعدادات من ملف الـ JSON المحلي.
