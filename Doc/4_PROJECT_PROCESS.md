## 4.1 قرارات المعمارية (Architectural Decisions - ADR)

### القرار رقم 1: اختيار نظام الملفات بدلاً من SQL

- **التاريخ:** فبراير 2026
- **القرار:** استخدام JSON/CSV لتخزين كافة البيانات.
- **السبب:** تسهيل النقل (Portability) وضمان الخصوصية وسهولة تكامل البيانات مع تطبيقات مثل Obsidian دون الحاجة لخادم قاعدة بيانات.

### القرار رقم 2: استخدام بايثون للمراقبة

- **التاريخ:** فبراير 2026
- **القرار:** إنشاء جسر (Bridge) مع Python لجلب الإحصائيات.
- **السبب:** مكتبة `psutil` في بايثون أكثر دقة وسهولة في التعامل مع موارد النظام العميقة مقارنة بـ Node.js وحده.
